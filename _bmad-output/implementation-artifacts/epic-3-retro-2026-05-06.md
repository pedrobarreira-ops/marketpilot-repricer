## What Went Well

### Pre-committed test scaffolds — new gold standard

82 tests were written and committed by the Epic-Start test-design subagent before a single line of implementation existed. Dev agents for Stories 3.1, 3.2, and 3.3 each walked in, read the ACs, looked at the pre-committed test files, and implemented to make them pass. Zero design ambiguity at implementation time. This is the cleanest handoff pattern to date and must become the default for every future epic.

The mock server (`tests/mocks/mirakl-server.js`) was also pre-committed — complete with fixture replay, failure injection, and capture API. Stories 3.2, 3.3, and every future Mirakl-touching story in Epics 4–7 inherit this infrastructure for free.

### Mirakl MCP gate — first confirmed production bug prevented

**The PC01 path catch is the first concrete proof that the Mirakl MCP mandate adds real value beyond process hygiene.**

During Story 3.2 implementation, the dev agent queried the Mirakl MCP live spec rather than guessing from training data. This surfaced that `/api/platform/configuration` is the correct PC01 path — the architecture distillate had an older `/api/configuration` placeholder. The bug would have caused HTTP 404 on every PC01 call in production. Caught at the Epic 3 layer, before any customer-facing code existed.

Canonical sentence for the record: *"The Mirakl MCP gate caught a production bug that would have caused 404 on PC01 for every live customer."*

### BAD pipeline self-corrected mid-epic — three fixes applied between PRs

Three BAD pipeline issues were identified from Story 3.1 observations and fixed before Stories 3.2/3.3 ran:

1. **Step 5 (sprint-status.yaml hallucination)**: Opus subagent was writing sprint-status.yaml on story branches, causing ~45k tokens of cleanup subagents. Fixed in the reference file. Confirmed clean for Stories 3.2 and 3.3.

2. **Step 6 (GitHub Actions skipped hallucination)**: Recurring across PRs #65 and #68 — CI status was drafted before CI had queued. Root cause: hallucination opportunity. Fix: CI status removed from PR body entirely (not "more careful" prompting — elimination). Held for PRs #69 and #70.

3. **Step 3 (Context7 pre-fetch)**: Pre-implementation doc-fetch step added for library reference hygiene. Mirakl MCP confirmed load-bearing for Stories 3.2/3.3 (see PC01 catch above).

The instinct to fix mid-batch rather than waiting for retro is correct. This should be a first-class operational pattern: when a pipeline fix is identified, apply it before the next story runs.

The `no-direct-fetch` ESLint rule is now mechanically enforcing the SSoT contract — no story can accidentally bypass `shared/mirakl/api-client.js` going forward.

### Step 5 before the fix: 45k tokens of cleanup cost

The sprint-status.yaml Opus write issue cost approximately 45k tokens in cleanup subagents during Story 3.1. That's close to two full context windows consumed resolving something that shouldn't have happened. The fix was applied promptly and confirmed clean for subsequent stories — but the pre-fix cost is real and worth registering.

**Reframe**: The fix arrived fast. The lesson is about detection speed, not about blaming the pipeline. The observation from Epic 3 is: pipeline fixes should be applied mid-batch as soon as identified, not deferred to retro. This happened correctly here.

### `project-context.md` over context-budget threshold

`node scripts/context-budget.js` flagged `project-context.md` at 669 lines (threshold: 600, over by +69 lines). The dominant section is Library Empirical Contracts + Operational Patterns at 178 lines — this section grows every epic as new contracts are verified and hoisted.

**Per CLAUDE.md per-epic retro checklist, this is a mandatory action item for Epic 3.** Surgical compression of the Library Empirical Contracts section before Epic 4 sharding starts.

### Source-file comment text scanner gotcha (new Library Empirical Contract)

The `no-direct-fetch` tests scan source code text for the substring `fetch(` — including in comments. Story 3.2's dev agent discovered this mid-implementation and had to rephrase comments from "calls fetch()" to "makes HTTP call" to avoid false positives. This will recur in any story that touches `shared/mirakl/` files and adds comments about fetch behavior.

**Action**: Add to project-context.md during the Item 1 compression pass as an explicit contract: *"Source-file comments in `shared/mirakl/` must NOT contain `fetch(` — the `no-direct-fetch` test suite scans comment text as well as code."*

## What Hurt — Reframed

### 1. Sentinel/forward-guard contradiction (Stories 4.3 + 4.8) — P0 BLOCKING

**Symptom:** `key.js` INSERTs `max_discount_pct = 0.0300` as a placeholder at PROVISIONING + the column is declared `NOT NULL` + `0.0300` is identical to the `15_plus` band mapping → Story 4.8's `IS NOT NULL` sentinel is unreachable for every DRY_RUN customer, and value-based disambiguation fails.

**Root cause:** Spec line 111 (Story 4.8) and Critical Constraint 11 declared `IS NOT NULL` as the forward-only sentinel, but the schema reality (NOT NULL column + placeholder INSERT from 4.3) the dev faithfully implemented contradicts that premise. Cross-story scope: touches `key.js`, the migration, and `margin.js`.

**Reframe:** This is a SPEC contradiction surfacing through Bundle B. The placeholder hack pre-dates Story 4.8 — it was always going to break the "first margin choice" sentinel; Bob's Story 4.8 shard didn't catch it because the contradiction was buried in the schema convention from Story 4.1 + 4.3. Pre-Epic-5 spec-vs-schema reconciliation is now a step Bob should add to sharding.

### 2. Migration drift now multi-source (3 issues compounded) — P0 BLOCKING

**Symptoms:**
- `20260430120730_create_audit_log_event_types.sql` (14-digit filename) blocks `npx supabase db push` entirely. The Supabase CLI sorted-merge can't reconcile it against `202604301207_create_baseline_snapshots.sql`. F5 amendment comment in the file reveals the intended name was `202604301207a_*`.
- Bundle B 4.3↔4.2 collision on `202604301211_create_scan_jobs.sql` — divergent contents in two worktrees during parallel BAD execution.
- Out-of-order `202605062200_story43_vault_and_scanjobs_rls.sql` created by Story 4.3 + applied directly to cloud DB (mid-Epic-4 hotfix) without a `schema_migrations` row.

**Reframe:** Filename convention is not currently load-bearing in BAD shard prompts — Bob shards stories without a migration-naming check, dev agents pick whatever timestamp seems plausible, parallel stories collide. The convention exists in distillate but isn't mechanically enforced. The 14-digit case (`20260430120730`) was a single typo (extra `30` in the seconds field) that no automation caught.

### 3. Bypass-aware middleware wrappers — anti-pattern propagating (3 routes now)

`authMiddlewareConditional` / `rlsContextConditional` / `releaseRlsClientConditional` invented in Story 4.7 for unit-test ergonomics (lets tests inject `req.user`/`req.db` mocks via global preHandler without spinning up Supabase). Pattern copy-pasted into 4.8 + 4.9. Three routes now embed a test-only escape hatch in production code.

**Risk:** if a future global preHandler ever sets `req.user` truthy before these wrapper hooks fire (e.g., a new auth-passthrough plugin in `server.js`), auth is silently skipped. Production never gets a loud failure.

### 4. Unit-test shadow-logic — tests drifting from their target

Increasing pattern across Stories 4.6 / 4.7 / 4.8: dev agents write unit tests that re-implement route helper functions locally rather than importing the real function under test. CI stays green during refactors. Worst case: Story 4.9's dashboard test (`tests/app/routes/dashboard/dry-run-minimal.test.js`) is **not in the npm test allowlist at all** — passes locally, doesn't run in CI.

**Reframe:** This is BAD subagent behavior — when the route module is hard to import under unit-test scaffolding (e.g., no Supabase, no full Fastify wiring), the agent shortcuts by reproducing the logic. The test still asserts behavior; it just asserts a *copy* of the behavior.

### 5. Bundle B + parallel BAD = filename collision risk

Stories 4.2 and 4.3 ran in parallel BAD worktrees both targeting `202604301211_create_scan_jobs.sql` with different contents. Caught in review (decision-needed flag), resolved manually. The migration-immutability rule doesn't fire mid-PR because each worktree thinks it's the first to claim the slot.

### 6. BAD Step 5/7 don't catch import-path errors

Story 4.3 shipped with `../../../shared/` (3 levels) when the route lives at depth 4. Both code-review (Step 5) and PR-review (Step 7) passed. Caught only by bad-review's CI-pending guard, fixed post-merge in commit `8eccdf6`. Memoed in `feedback_step5_import_path_boot_check.md` — `node --check` boot test would catch this in seconds. Same depth bug recurred in Story 4.6's ATDD scan-failed stub (commits `438d69a` + `b9f1a6e`).

### 7. BAD Step 6 PR-body hallucinations — 9 in a row

Every Epic 4 PR (#71-#79) overstated file counts, listed `sprint-status.yaml` or `dependency-graph.md` as changed when absent from diff, or mis-stated insertion counts. No functional impact. Trust erosion is real — review reads the body to orient before reading the diff.

### 8. Context7 not being called (Epic 3 retro Item 4 watch trigger fired)

Zero Context7 references across all 9 Epic 4 story files. The instruction was added in Epic 3 prep but dev agents aren't invoking it. Mirakl MCP carries the API-spec load; Context7 was meant for library docs (Stripe/Resend/pg/etc.). Epic 4's library surface was minimal (only Resend in 4.6, where SDK contract is trivial) — so the failure may be context-appropriate, not a pipeline failure.

## Key Takeaways

1. **Spec-vs-schema reconciliation belongs in sharding** — Story 4.8's sentinel works only if Story 4.3's schema convention permits it. Bob's shard for 4.8 inherited a contradiction buried in 4.1's `NOT NULL` + 4.3's placeholder INSERT. Add a "schema-reality check" step to BAD sharding for any story that depends on column nullability semantics.

2. **Mirakl MCP value compounds per endpoint touched** — 5 schema-shape corrections in Story 4.4 alone, all production bugs caught at impl time. The gate's ROI grows; treat MCP queries as mandatory not advisory.

3. **Atomicity bundle gates work in BAD** — Bundle B's `merge_blocks` held Story 4.3 until Story 4.4 landed despite earlier PR open date. Bundle C at Epic 5+ inherits a proven mechanism.

4. **Live smoke is now load-bearing** — 2 real bugs found, neither reachable through fixture tests. Pre-merge smoke is mandatory for any Mirakl-touching code. Make it a Story 7.8 gate dependency.

5. **Recurring patterns earn pipeline patches, one-offs don't** — 9 PR-body hallucinations is a documented pattern but cosmetic only → accept-and-monitor. 2 import-path bugs in 1 epic is a recurring functional bug → `node --check` boot test patch (Item 4). The threshold is "harm or rate," not "anything that can be fixed."

6. **Test quality requires guardrails, not virtue** — shadow-logic anti-pattern propagated across 3 stories without anyone noticing. CI was green throughout. Mechanical guardrails (import-the-function rule, npm test allowlist enforcement) > review attention.

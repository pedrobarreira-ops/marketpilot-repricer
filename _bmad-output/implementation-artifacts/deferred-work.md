## Deferred from: code review of 1-2-envelope-encryption-module-master-key-loader-secret-scanning-hook (2026-05-01)

Theoretical concern for very long base64 (>76 chars where openssl wraps); 32-byte keys produce 44 chars, so never triggered by the documented `openssl rand -base64 32`. Defense-in-depth fix would be to strip all whitespace before decoding.
- **secret-scanner pattern coverage is narrow (5 patterns)** — only catches MASTER_KEY-style assignment, shop_api_key assignment, sk_live_, sk_test_, Authorization Bearer. Real Stripe/GitHub leak shapes (restricted-scope keys with dots, scoped tokens) and Supabase service-role JWTs (`eyJ...`) are not covered. Fixing requires a more elaborate AST pattern or a custom rule (Story 3.1+ ships custom rules); not justified at MVP scale.

## What Hurt — Reframed

Bob's first read framed five themes as story-level pain. Pedro's reframe surfaces the underlying systemic patterns:

### 1. Library empirical-contract surprises are a CATEGORY problem, not a stories problem

**Bob's first frame**: "4 of 5 Epic 1 stories had a library contract surprise."

**Reframe**: Every external-library integration surfaces at least one contract violation. This will continue through Epic 4 (Stripe), Epic 6 (PRI01), Epic 11 (Stripe webhooks), and any other SDK we touch. The retro should NOT treat this as Epic 1 noise — it is a permanent property of integration work that requires a systemic answer (the `integration_test_required: true` story-level tag + Phase 4.5 BAD gate).

### 2. "Spec-verbatim regex transcription bugs" misframes the cause

**Bob's first frame**: "Story 1.2 had two transcription bugs in the source distillate AC#4."

**Reframe**: ACs containing literal regex / SQL / code-like content are a brittle AC category. The fix is not back-porting Story-1.2-specific regexes — it is adding a distillation pass that validates regex/SQL ACs against examples. Future epics with code-in-AC will hit this again (Epic 6 has CSV format ACs; Epic 9 has audit-log SQL ACs).

### 3. Pool lifecycle "deferred 5x" reads as a hurt; it's actually working-as-designed

**Bob's first frame**: "Pool lifecycle / SIGTERM deferred 5 times."

**Reframe**: SSoT-deferred is the whole point. The risk is NOT the deferral — the risk is that Story 2.1 misses one pool. Reframe in retro as a **win with an Epic 2 vigilance flag**: Story 2.1 spec must explicitly enumerate all three pools to absorb (`health.js`, `heartbeat.js`, `founder-admin-only.js`).

### 4. Local Supabase fragility is underweighted

**Bob's first frame**: "Cost a session in Story 1.4."

**Reframe**: Pedro's branch now permanently carries a `config.toml` with 6 services disabled (`[realtime]`, `[studio]`, `[storage]`, `[edge_runtime]`, `[analytics]`, plus baseline). That is **branch tech debt**. When Epic 8 dashboard needs realtime, or Epic 4 storage needs file uploads, re-enabling will require another triage cycle. Track as a **known future-cost on the watch list**, not a one-time hit.

### 5. Cross-file test parallelism race isn't latent — it's actively wrong

**Bob's first frame**: "Latent until CI is wired."

**Reframe**: It already fails on Pedro's box when running `node --test tests/integration/signup-flow.test.js tests/integration/admin-middleware.test.js` in one invocation. The chore PR fixes it as a **P0**, not "queue it." Escalated by becoming load-bearing for Phase 4.5 (the integration-test gate needs `npm run test:integration` to exist).

### 6. BAD/bad-review customization velocity isn't called out

**New observation surfaced in Pedro's reframe**: ~377 lines of skill customization shipped during Epic 1 (documented in `_bmad-output/implementation-artifacts/bad-customization-notes.md`). Items shipped: Worker-Path Opus Gate, Mirakl repointing, mcp-forbidden-patterns, merge_blocks reads, Phase 5.5 manual smoke. Items pending: Phase 4.5 integration-test gate.

**Risk**: rate of accumulation is itself a thing to watch. If upstream BAD ever updates, the three-way merge gets hard. Worth flagging as a **watch item**, not a hurt yet.

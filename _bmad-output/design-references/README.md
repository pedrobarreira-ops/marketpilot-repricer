## How to view a screen

**Live visual (Phases B + C)**: open [`bundle/project/dashboard-and-audit.html`](bundle/project/dashboard-and-audit.html) in a browser. The canvas loads with all 13 artboards visible. Use the **tweaks toolbar** (top-right of the canvas) to navigate between them.

**Reference stub (any of 01–15)**: open the corresponding [`screens/0X-*.html`](screens/) file. You'll see the skeleton-section + FR traceability + a link to the bundle for the live visual.

## How to reference a screen from other docs

Cite by the stable stub filename:

> **From Architecture**: "Circuit-breaker trip surfaces via the dashboard banner system (see `_bmad-output/design-references/screens/03-dashboard-live.html` and `screens/06-audit-log-root.html` Atenção feed)."
>
> **From Epics**: "Story 5.3 — Audit log search-by-SKU. Implements `screens/12-audit-search-active.html` per skeleton §4.1.4 (UX10), FR38."
>
> **From Stories / BAD subagents**: when implementing a screen, first open the stub for skeleton + FR context, then open the bundle canvas for the live visual.

The stub filenames are stable. They work for both already-rendered (01–12, 06b — link out to the bundle) and pending (13–15 — Phase D placeholder) screens. References don't break when Phase D lands.

## What this folder does NOT contain

- **Implementation code** — lives in `app/`, `worker/`, etc. (per `_bmad-output/planning-artifacts/prd.md` topology).
- **Design intent / decision rationale** — lives in [`free-report-design-chat.md`](free-report-design-chat.md) (free-report visual-DNA decisions) and [`bundle/chats/chat2.md`](bundle/chats/chat2.md) (repricer Phase B/C decisions).
- **Authoritative behavior contract** — lives in [`../planning-artifacts/prd.md`](../planning-artifacts/prd.md).
- **PT microcopy verbatim** — lives in [`../planning-artifacts/ux-skeleton.md`](../planning-artifacts/ux-skeleton.md) §9.

If you need to know **what** a screen does, read the PRD. If you need to know **why** a design choice was made, read the design-intent transcripts and the UX skeleton. If you need to know **how it should look**, open the bundle canvas (Phases B + C) or wait for Phase D.
